# Full Stack Open - Part 0: Fundamentals of Web Apps

This repository contains my solutions for the **Part 0** of the Full Stack Open course from the University of Helsinki. 

The focus of these exercises is to understand how web applications work by visualizing the communication between the **browser** and the **server** through sequence diagrams.

### 📚 Learning Objectives:
- Understanding HTTP GET and POST requests.
- Analyzing the traditional "form submission" flow with server-side redirects (302).
- Exploring the differences between traditional web apps and **Single Page Applications (SPA)**.
- Learning how AJAX/JavaScript fetches data in the background without reloading the page.

### 🛠️ Exercises:

- **[Exercise 0.4: New Note Diagram](./04_New_note_diagram.md)**  
  *Visualizes the full page reload cycle when a new note is added in a traditional web app.*

- **[Exercise 0.5: Single Page App Diagram](./05_Single_page_app_diagram.md)**  
  *Shows the initial loading process of a Single Page Application (SPA).*

- **[Exercise 0.6: New Note in SPA Diagram](./06_New_note_in_Single_page_app_diagram.md)**  
  *Demonstrates how a SPA handles data submission locally via JavaScript without refreshing the page.*
